# M1.1e — Rebase STL-wrap template on Stock_U1_Blank.3mf

> Builds on M1.1. STL-wrap path ONLY. TDD, task-by-task.

**Goal:** Generated STL→U1 3MF opens in Snapmaker Orca with **no** "Customized Preset" dialog, by basing the bundled
project_settings template on the clean stock single-colour U1 export instead of the Bambu-derived Shadow_AMS.

**Root cause (proven across exp1–exp8):** the dialog is driven by the project's **filament/printer preset VALUES**
differing from the installed stock system presets (61 value diffs remain even after matching ids, clearing gcode,
adding model_settings/slice_info, etc.). Single-key patches cannot fix a whole-preset mismatch. Stock_U1_Blank.3mf
is a clean 1-filament U1 project that opens with no dialog — use its project_settings verbatim.

**Why this is allowed:** an STL has no source slicer settings to preserve, so adopting stock U1 defaults is correct,
not lossy. (Contrast: the 3MF *repair* path must preserve the source's own settings — it is NOT touched here.)

## Scope / guardrails
- STL-wrap path only (`stl_wrap.py` + its bundled template).
- Do NOT touch: 3MF repair (`repair.py`), optimization profiles (`optimize.py`), geometry generation
  (`stl_io.py`, `build_model_xml`, `bed_center_transform`), the U1 profile system (`profile.py`).
- Keep 1 filament; per-filament arrays already consistent in stock blank (all length 1).
- `--opt-profile` for STL is OUT OF SCOPE here (STL path currently ignores it; wiring it is a separate task — flagged).

## Stock blank facts (verified)
572 keys · `printer_model="Snapmaker U1"` · `gcode_flavor="klipper"` · `printer_settings_id="Snapmaker U1 (0.4 nozzle)"`
· `print_settings_id="0.20 Standard @Snapmaker U1 (0.4 nozzle)"` · `filament_settings_id=["Generic PLA"]`
· `filament_vendor=["Generic"]` · no `different_settings_to_system` key · `filament_start_gcode=[""]`,
`filament_end_gcode=["\n"]` · all per-filament arrays length 1 · `nozzle_diameter` length 4 (per-extruder).

```python
def _base_settings(profile_name: str = "snapmaker_u1") -> dict:
    # Base template IS a clean stock single-colour Snapmaker U1 export (Stock_U1_Blank.3mf):
    # correct U1 identity, stock preset ids, empty filament gcode, consistent 1-filament arrays.
    # Use it verbatim so Orca matches installed system presets (no "Customized Preset" dialog).
    cfg = json.loads((files("snapstudio_core.data.templates")

    conform_filament_arrays(cfg, 1)   # invariant guard; no-op on the stock 1-filament base
    return cfg
```
Remove now-unused imports: `apply_swap`, `load_profile`, `apply_clamps`, `load_rules`. Keep
`conform_filament_arrays`, `json`, `copy`, `files`. (`profile_name` kept for API compatibility, unused.)
